# Functional Test Suite Design — TDD Red-Green via Claude Code CLI

**Date:** 2026-02-28
**Status:** Design
**Branch:** `main`

---

## 1. Overview

Build a comprehensive functional test suite for Virtual Analyst that:

1. Uses the **26-chapter user manual** (`docs/user-manual/`) as the specification
2. Follows **TDD Red-Green** discipline — each test encodes a manual claim; RED means the app diverges from the spec, GREEN means it conforms
3. Runs sequentially via **`claude -p "..."`** prompts from shell scripts
4. Drives a real browser using **Playwright** against the dev server (`localhost:3000`)
5. Seeds test data via API calls before execution and cleans up after

---

## 2. TDD Red-Green Methodology

### The Cycle

```
Manual Chapter → Playwright Test → Run Test → RED or GREEN
```

| Phase | Meaning |
|-------|---------|
| **RED** | Test fails because the app does NOT match what the manual documents. This is a specification violation — either the code or the manual needs updating. |
| **GREEN** | Test passes because the app behavior matches the manual specification exactly. The feature works as documented. |

### Why TDD for Functional Tests?

- The **user manual IS the specification**. Each chapter defines expected behavior.
- Each test **encodes one or more claims** from a specific chapter.
- Running the suite answers: **"Does the app do what the manual says it does?"**
- Failed tests create a traceable link between documentation and implementation gaps.
- New features get manual chapters FIRST (spec), then tests (RED), then implementation (GREEN).

## 4. Prompt Template

Every `claude -p` invocation follows this structure:

```bash
claude -p "
You are executing a TDD RED-GREEN functional test for Virtual Analyst.

SPECIFICATION (from docs/user-manual/chXX-feature-name.md):
<< exact excerpt from the manual describing expected behavior >>

TASK:
1. Write a Playwright test in TypeScript that verifies this behavior
   against http://localhost:3000
2. Save the test to tests/functional/chXX-test-name.spec.ts
3. Install Playwright if needed: npx playwright install chromium
4. Run the test: npx playwright test tests/functional/chXX-test-name.spec.ts
5. Report the result in this EXACT format on the last line:
   RESULT: GREEN — chXX-test-name — PASS
   or
   RESULT: RED — chXX-test-name — FAIL — <reason>

TEST CONSTRAINTS:
- Base URL: http://localhost:3000
- Test user: functional-test@va.dev / TestPass123!
- Use page.waitForSelector() or expect(locator).toBeVisible() for async content
- Assert on visible UI text and elements, not CSS classes or data attributes
- Each test must be independently runnable
- Do not modify application source code
- If the test needs seeded data, use IDs from tests/functional/fixtures/test-constants.ts
"
```

## 5. Test Seeding Strategy

### seed-test-data.sh

Seeds the dev database with predictable test fixtures via API calls:

```bash
#!/usr/bin/env bash
# Seed test data for functional test suite
# Requires: dev server running at localhost:8000 (API) and localhost:3000 (web)

BASE_URL="http://localhost:8000/api"

# 1. Create test user (via Supabase auth or direct DB insert)
# Store auth token for subsequent calls

# 2. Create test organization/tenant
# POST /api/tenants → store TENANT_ID

# 3. Seed a baseline from marketplace template
# POST /api/baselines → store BASELINE_ID

# 4. Create a draft from the baseline
# POST /api/drafts → store DRAFT_ID

# 5. Execute a run on the draft
# POST /api/runs → store RUN_ID

# 6. Create an AFS engagement
# POST /api/afs/engagements → store AFS_ENGAGEMENT_ID

# 7. Create a budget
# POST /api/budgets → store BUDGET_ID

# 8. Create a covenant monitor
# POST /api/covenants → store COVENANT_ID

# 9. Create a board pack
# POST /api/board-packs → store BOARD_PACK_ID

# 10. Write all IDs to tests/functional/fixtures/test-constants.ts
cat > tests/functional/fixtures/test-constants.ts << EOF
export const TEST_USER = {
  email: 'functional-test@va.dev',
  password: 'TestPass123!',
};
export const SEEDED_IDS = {
  tenantId: '${TENANT_ID}',
  baselineId: '${BASELINE_ID}',
  draftId: '${DRAFT_ID}',
  runId: '${RUN_ID}',
  afsEngagementId: '${AFS_ENGAGEMENT_ID}',
  budgetId: '${BUDGET_ID}',
  covenantId: '${COVENANT_ID}',
  boardPackId: '${BOARD_PACK_ID}',
};
EOF

echo "Test data seeded. IDs written to tests/functional/fixtures/test-constants.ts"
```

### cleanup-test-data.sh

Removes test fixtures in reverse order to avoid FK violations:

```bash
# DELETE board pack, covenant, budget, run, draft, baseline, AFS engagement, tenant, user
# Uses the same IDs from test-constants.ts
```

## 8. Helper Library

### lib/test-helpers.sh

```bash
#!/usr/bin/env bash
# Shared functions for the functional test runner

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
CYAN='\033[0;36m'
NC='\033[0m' # No Color
BOLD='\033[1m'

PASS_COUNT=0
FAIL_COUNT=0
SKIP_COUNT=0
RESULTS=()

log_phase() {
    echo ""
    echo -e "${CYAN}${BOLD}════════════════════════════════════════════════════${NC}"
    echo -e "${CYAN}${BOLD}  $1${NC}"
    echo -e "${CYAN}${BOLD}════════════════════════════════════════════════════${NC}"
    echo ""
}

log_phase_complete() {
    echo ""
    echo -e "${CYAN}  ✓ $1 complete${NC}"
    echo ""
}

run_tdd_test() {
    local test_name="$1"
    local spec="$2"

    echo -e "${YELLOW}  ▶ Running TDD test: ${test_name}${NC}"

    local prompt="You are executing a TDD RED-GREEN functional test for Virtual Analyst.

${spec}

TEST CONSTRAINTS:
- Base URL: http://localhost:3000
- Test user: functional-test@va.dev / TestPass123!
- Use page.waitForSelector() or expect(locator).toBeVisible() for async content
- Assert on visible UI text and elements, not implementation details
- Each test must be independently runnable
- Do not modify application source code
- If the test needs seeded data, import IDs from tests/functional/fixtures/test-constants.ts
- Report the result on the LAST LINE in this EXACT format:
  RESULT: GREEN — ${test_name} — PASS
  or
  RESULT: RED — ${test_name} — FAIL — <one-line reason>"

    local output
    output=$(claude -p "$prompt" 2>&1) || true

    # Save full output to results directory
    echo "$output" > "$(dirname "$0")/results/${test_name}.log"

    # Parse the RESULT line
    local result_line
    result_line=$(echo "$output" | grep "^RESULT:" | tail -1)

    if echo "$result_line" | grep -q "GREEN"; then
        echo -e "    ${GREEN}✓ GREEN — ${test_name} — PASS${NC}"
        PASS_COUNT=$((PASS_COUNT + 1))
        RESULTS+=("GREEN|${test_name}")
    elif echo "$result_line" | grep -q "RED"; then
        local reason
        reason=$(echo "$result_line" | sed 's/.*FAIL — //')
        echo -e "    ${RED}✗ RED — ${test_name} — FAIL — ${reason}${NC}"
        FAIL_COUNT=$((FAIL_COUNT + 1))
        RESULTS+=("RED|${test_name}|${reason}")
    else
        echo -e "    ${YELLOW}? SKIP — ${test_name} — No result line found${NC}"
        SKIP_COUNT=$((SKIP_COUNT + 1))
        RESULTS+=("SKIP|${test_name}|No result line")
    fi
}

print_summary() {
    echo ""
    echo -e "${BOLD}════════════════════════════════════════════════════${NC}"
    echo -e "${BOLD}  FUNCTIONAL TEST SUMMARY${NC}"
    echo -e "${BOLD}════════════════════════════════════════════════════${NC}"
    echo ""
    echo -e "  ${GREEN}GREEN (pass): ${PASS_COUNT}${NC}"
    echo -e "  ${RED}RED (fail):   ${FAIL_COUNT}${NC}"
    echo -e "  ${YELLOW}SKIP:         ${SKIP_COUNT}${NC}"
    echo -e "  Total:        $((PASS_COUNT + FAIL_COUNT + SKIP_COUNT))"
    echo ""

    if [ ${FAIL_COUNT} -gt 0 ]; then
        echo -e "${RED}${BOLD}  RED Tests (specification violations):${NC}"
        for result in "${RESULTS[@]}"; do
            if [[ "$result" == RED* ]]; then
                local name reason
                name=$(echo "$result" | cut -d'|' -f2)
                reason=$(echo "$result" | cut -d'|' -f3)
                echo -e "    ${RED}✗ ${name}: ${reason}${NC}"
            fi
        done
        echo ""
    fi

    if [ ${FAIL_COUNT} -gt 0 ]; then
        echo -e "${RED}${BOLD}  EXIT CODE: 1 (specification violations found)${NC}"
        exit 1
    else
        echo -e "${GREEN}${BOLD}  EXIT CODE: 0 (all tests match specification)${NC}"
        exit 0
    fi
}

export -f run_tdd_test log_phase log_phase_complete print_summary
export PASS_COUNT FAIL_COUNT SKIP_COUNT RESULTS
```

## 9. Master Runner

### run-all.sh

```bash
#!/usr/bin/env bash
# run-all.sh — Master runner for TDD Red-Green functional test suite
#
# Usage:
#   ./scripts/functional-tests/run-all.sh              # Run all phases
#   ./scripts/functional-tests/run-all.sh --phase 2    # Run only phase 2
#   ./scripts/functional-tests/run-all.sh --no-seed    # Skip seeding
#
# Prerequisites:
#   - Dev server running at localhost:3000 (web) and localhost:8000 (API)
#   - Claude Code CLI installed and authenticated
#   - Node.js and npx available

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
source "$SCRIPT_DIR/lib/test-helpers.sh"

PHASE_FILTER=""
SEED=true

while [[ $# -gt 0 ]]; do
    case $1 in
        --phase) PHASE_FILTER="$2"; shift 2 ;;
        --no-seed) SEED=false; shift ;;
        *) echo "Unknown option: $1"; exit 1 ;;
    esac
done

echo -e "${BOLD}╔══════════════════════════════════════════════════════╗${NC}"
echo -e "${BOLD}║  Virtual Analyst — TDD Red-Green Functional Tests   ║${NC}"
echo -e "${BOLD}║  Specification: docs/user-manual/ (26 chapters)     ║${NC}"
echo -e "${BOLD}╚══════════════════════════════════════════════════════╝${NC}"

# Ensure Playwright is installed
if ! npx playwright --version &>/dev/null; then
    echo "Installing Playwright..."
    npm init playwright@latest -y
    npx playwright install chromium
fi

# Create results directory
mkdir -p "$SCRIPT_DIR/results"

# Seed test data
if $SEED; then
    echo ""
    echo "Seeding test data..."
    bash "$SCRIPT_DIR/seed-test-data.sh"
fi

# Run phases
run_phase() {
    local phase_num="$1"
    local phase_script="$SCRIPT_DIR/phase-${phase_num}-*.sh"

    if [[ -n "$PHASE_FILTER" && "$PHASE_FILTER" != "$phase_num" ]]; then
        return
    fi

    for script in $phase_script; do
        if [[ -f "$script" ]]; then
            bash "$script"
        fi
    done
}

run_phase 1
run_phase 2
run_phase 3
run_phase 4
run_phase 5
run_phase 6

# Cleanup
if $SEED; then
    echo ""
    echo "Cleaning up test data..."
    bash "$SCRIPT_DIR/cleanup-test-data.sh" || true
fi

# Print final summary
print_summary
```

## 10. Playwright Configuration

### tests/functional/playwright.config.ts

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: '.',
  timeout: 30_000,
  expect: { timeout: 10_000 },
  fullyParallel: false,        // Sequential — matches phase execution order
  retries: 0,                   // TDD: no retries, RED means RED
  reporter: [['list'], ['json', { outputFile: 'results.json' }]],
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
  ],
});
```
